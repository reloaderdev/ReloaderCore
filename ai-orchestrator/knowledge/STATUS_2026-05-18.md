## Ecosistema completo — idea y responsabilidades

### ReloaderCore — Team Leader
**Idea:** orquestador global, no técnico. Conoce a todos los agentes, descompone tareas,
define el orden de ejecución y valida que los resultados sean coherentes.
**No escribe código, SQL ni configuración.**

| Responsabilidad | Detalle |
|----------------|---------|
| Conocer el ecosistema | Mapa de agentes en ECOSYSTEM.md |
| Descomponer tareas | Analizar qué agentes se ven afectados |
| Orden de ejecución | DB → Backend → Mobile (siempre) |
| Delegar con contexto | Pasa output del agente anterior al siguiente |
| Validar resultados | Verifica coherencia entre agentes |
| Bloquear y escalar | Para todo si hay ambigüedad, pregunta al usuario |

---

### ReloaderDB — Agent DBA
**Idea:** dueño exclusivo de la base de datos. Cualquier proyecto se conecta a ella,
ninguno la modifica directamente. Siempre el primero en ejecutarse.

| Responsabilidad | Detalle |
|----------------|---------|
| Migraciones | Flyway V000→VXxx, nunca modificar aplicados |
| Stored Procedures | Toda la lógica SQL vive aquí |
| Seeds | Datos iniciales y maestros |
| Docker local | SQL Server para desarrollo |
| Independencia | REST, MS, Mobile, y futuros proyectos solo configuran conexión |

---

### reloaderproject-rest — Agent REST
**Idea:** backend principal. Expone endpoints JAX-RS consumidos por Mobile y otros clientes.
Arquitectura estricta de 3 capas.

| Responsabilidad | Detalle |
|----------------|---------|
| Endpoints REST | JAX-RS: Resource → Service → DAO |
| Autenticación web | Servlets: LoginServlet, RegisterServlet |
| Deploy | Docker → Azure Container Registry → Azure Web App |
| Dependencia | Necesita ReloaderDB listo antes de cualquier cambio |

---

### reloaderproject-ms- — Agent Microservicios
**Idea:** servicios independientes en Kotlin/Ktor para operaciones específicas.
Cada microservicio es autónomo y deployable por separado.

| Responsabilidad | Detalle |
|----------------|---------|
| auth-service | Registro de usuarios públicos |
| supplier-service | Registro de proveedores |
| Deploy | Docker independiente por servicio |
| Dependencia | Necesita ReloaderDB listo antes de cualquier cambio |

---

### ReloaderGames — Agent Mobile
**Idea:** app Kotlin Multiplatform que consume REST y Microservicios.
Siempre el último en ejecutarse — depende de que los backends estén listos.

| Responsabilidad | Detalle |
|----------------|---------|
| Screens | Compose Multiplatform, Android + iOS |
| Consumo de APIs | Ktor Client, repositorios, ViewModels |
| Publicación | Play Store |
| Dependencia | Necesita REST y MS listos antes de implementar UI |

---

## Orden de ejecución del ecosistema

```
ReloaderDB → reloaderproject-rest / reloaderproject-ms- → ReloaderGames
   (DB)              (Backend)                                (Mobile)
```
